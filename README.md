# Telegram-бот для личного тайм-менеджмента

MVP Telegram-бота, который помогает не откладывать задачи. Основной UX теперь свободный текст: пользователь пишет как в обычном чате, а бот сам определяет намерение, создает задачу, переносит ее, закрывает или отвечает как accountability-партнер.

Теперь в проекте есть опциональный AI-ассистент: accountability-партнер, который помогает начать, вернуться в фокус, получить короткий план, вспомнить «ради чего» и не зависнуть в бесконечном «сейчас начну».

## Возможности

- `/start` с главным меню.
- Добавление задачи через пошаговый сценарий.
- Свободный текст как основной способ управления.
- Intent detection: создание задач, обычный чат, прокрастинация, помощь, управление задачами, эмоции, мотивация.
- Быстрый ввод: `Напомни через 20 минут доделать диплом`.
- Повторные напоминания о старте.
- Кнопки `Начал`, `Отложить на 10 минут`, `Готово`, `Отменить`.
- Проверка прогресса через 25 минут после старта.
- Список активных задач, история закрытых задач и активные напоминания.
- SQLite-хранилище пользователей, задач, напоминаний, событий и AI-контекста.
- AI-кнопки внутри задач и AI-команды.
- Fallback-шаблоны без OpenAI API.
- Проактивные AI-nudge сообщения по триггерам без спама.

## AI-режим

OpenAI API не обязателен. По умолчанию `AI_OPENAI_ENABLED=false`, поэтому бот использует локальные fallback-сообщения из `services/fallback_messages.py` и локальные правила intent detection.

Чтобы включить OpenAI:

```env
AI_OPENAI_ENABLED=true
OPENAI_API_KEY=sk-...
OPENAI_CLASSIFIER_MODEL=gpt-5-nano
OPENAI_SIMPLE_MODEL=gpt-5-nano
OPENAI_COMPLEX_MODEL=gpt-5-mini
```

Если OpenAI недоступен, вернул ошибку, таймаут или лимит, бот логирует ошибку и автоматически отвечает fallback-шаблоном. Основные напоминания и задачи от OpenAI не зависят.

Когда `AI_OPENAI_ENABLED=true`, свободный текст сначала идет в OpenAI action planner. Он решает, какое действие вызвать и какие параметры передать: создать одну или несколько задач, ответить как чат, показать задачи, перенести задачу или включить анти-прокрастинационный сценарий. Если OpenAI недоступен, бот откатывается на локальные правила.

Модели разделены по стоимости:

- `OPENAI_CLASSIFIER_MODEL=gpt-5-nano` - дешевый классификатор намерений.
- `OPENAI_SIMPLE_MODEL=gpt-5-nano` - простые chat-ответы.
- `OPENAI_COMPLEX_MODEL=gpt-5-mini` - советы по задачам, планы, разборы и сложные ответы.

Задача создается из AI-классификации только при уверенности не ниже `OPENAI_CREATE_TASK_CONFIDENCE_THRESHOLD`, чтобы обычные вопросы не превращались в задачи.
В одном сообщении можно поставить несколько задач, например: `добавь завтра: в 9 проснуться, в 9:15 зарядка, в 10 учеба`.

AI ведет себя как активный участник процесса:

- может уточнить, если сообщение непонятное;
- предлагает следующий маленький шаг;
- возвращает в фокус во время активной задачи;
- может написать первым после старта задачи или после повторных переносов;
- не спамит: проактивные сообщения проходят через cooldown и отключаются командой `/ai_off`.

## Свободный текст

Можно писать без команд:

```text
надо сделать диплом
напомни через 20 минут заняться бизнесом
в 15:00 сделать задачу по проекту
сколько сейчас времени
объясни коротко, что такое фокус-блок?
не могу начать
очень много всего
с чего начать диплом
помоги разбить
я начал
готово
перенеси на 30 минут
дай пинок
напомни зачем
```

Бот не спрашивает «что вы хотите сделать?». Он классифицирует сообщение и выбирает действие сам. Кнопки и команды остаются запасным способом управления.

По умолчанию свободный текст считается обычным bro-chat сообщением. Задача создается только при явном намерении: `надо...`, `нужно...`, `сделать...`, `напомни...`, `добавь задачу...`. Поэтому фразы вроде `расскажи что-нибудь интересное` или `старых задач именно` не превращаются в новые задачи.

Время берется из `BOT_TIMEZONE`. Если указать время раньше текущего времени в этой зоне, бот не создаст задачу и попросит выбрать время позже.

`/tasks` и кнопка «Мои задачи» показывают только активные задачи. Завершенные и отмененные задачи остаются в базе как история и доступны через `/history` или кнопку «История задач».

Обычные вопросы идут в режим general chat. С OpenAI бот отвечает ближе к ChatGPT, но коротко и в стиле accountability-партнера. Без OpenAI есть локальные ответы на базовые вопросы, например текущее время, дату, приветствие и справку по возможностям.

## AI-команды

- `/help_task [id]` - совет по задаче.
- `/motivate [id]` - короткое мотивационное сообщение.
- `/focus_me [id]` - вернуть в рабочее состояние.
- `/why [id]` - напомнить, ради чего пользователь работает.
- `/boost [id]` - мотивационный трек плюс короткий пинок.
- `/start_step [id]` - самый маленький следующий шаг.
- `/panic [id]` - быстро собраться, когда хаос.
- `/tone bro` - выбрать стиль. Доступно: `supportive`, `balanced`, `tough`, `bro`.
- `/ai_on` и `/ai_off` - включить или выключить AI-поддержку для пользователя.

Если `id` не указан, бот берет последнюю активную задачу.

## Цели, фразы и треки

Цели и причины:

```text
/goal add закончить диплом
/goals
```

Персональные мотивационные формулировки:

```text
/phrase add Я делаю это ради нормальной жизни
/phrase
```

Треки:

```text
/track add start Название трека | https://example.com
/track add focus Название | описание без ссылки
/tracks
```

Категории треков: `start`, `focus`, `comeback`, `push`, `finish`.

## Жизненный цикл задачи

- `pending` - задача создана и ждет первого напоминания.
- `nudging` - бот напоминает о старте и повторяет напоминания.
- `snoozed` - пользователь отложил напоминание.
- `in_progress` - пользователь подтвердил старт.
- `done` - задача завершена.
- `cancelled` - задача отменена.

Если задача переносилась 3+ раза, AI становится прямее и чаще возвращает к действию, но без стыда и агрессии.

## OpenAI-архитектура

- `services/ai_service.py` - опциональный OpenAI SDK + fallback при ошибках.
- `services/ai_system_rules.py` - системные правила характера и стиля AI-помощника.
- `services/intent_service.py` - классификация свободного текста и выбор действия.
- `services/prompt_builder.py` - компактный промт из задачи, статуса, переносов, целей, фраз и режима тона.
- `services/user_context_service.py` - сбор пользовательского контекста.
- `services/motivation_service.py` - сценарии AI и анти-прокрастинационные триггеры.
- `services/tracks_service.py` - мотивационные треки.
- `services/fallback_messages.py` - локальные шаблоны, когда OpenAI выключен или недоступен.
